## 一、问题陈述

剧本杀门店日常运营面临三件事的协调：

1. **剧本** → 店里有哪些本、每个本要几个人、每个角色什么时候进场
2. **卡司（DM/主持）** → 谁会带哪个本、谁擅长哪个角色、当天有没有空
3. **房间** → 哪个房间今天空着、能不能塞下这场

这三件事靠脑子记、表格记、微信群吼，效率低，容易撞车出错。

---

## 二、目标

做一个**一人门店也能用**的排期管理工具，覆盖：

- 排期创建（选剧本、选房间、选卡司、选时间）
- 冲突检测（同一卡司不能同时带两个本）
- 玩家签到（扫码选角色）
- 评价收集（玩完了扫码打分）
- 会员管理（客户档案、余额）
- 基本数据统计（哪些本上座率高）

---

## 三、目标用户

| 用户 | 场景 |
|------|------|
| **门店老板/店长** | 创建排期、管理房间卡司、看统计数据 |
| **DM/卡司** | 被分配排期、查看自己当天排班 |
| **玩家** | 扫码签到选角色、扫码评价 |

> 优先级：第一版只做**门店老板视角**（管理后台），玩家端只做签到和评价两个公开页。

## 六、技术决策

| 决策点 | 结论 |
|--------|------|
| 技术栈 | Express + SQLite + React + TS + Tailwind（已定，不改） |
| 数据库余额单位 | **分**（整数），不用元（避免浮点问题） |
| 前后端分离 | 开发用 Vite proxy，生产用 Express serve 静态文件（已定） |
| 单仓库 | client + server + data 共存（已定，不改） |
| 后端拆分 | index.ts 和 db.ts 都超过 1000 行，建议按模块拆分路由，但不影响功能，先跑起来再说 |
