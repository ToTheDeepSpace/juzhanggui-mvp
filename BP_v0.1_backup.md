> **项目代号**：剧掌柜  
> **版本**：BP v0.1（内部初稿）  
> **日期**：2026-04-26

---

## 一、这是一件什么事

做一个覆盖剧本杀行业全产业链的数字化运营平台。

把发行商、店家、卡司/ DM、玩家四方连在一起——
让发行商看到真实的市场数据，让店家管得轻松，
让卡司有尊严有成长，让玩家拼本更方便。

---

## 二、为什么现在是个机会

### 剧本杀行业还在扩张，但工具很落后

- 全国剧本杀门店约 5-10 万家，年市场规模超 200 亿
- 大量门店还在用 Excel + 微信群管排期、管卡司、管物资
- 卡司/DM 普遍被当"耗材"用，高离职率是行业痛点
- 没有一家平台同时拥有发行商、店家、卡司、玩家的完整数据

### 技术条件已经成熟

- 中医 SaaS、餐饮 SaaS 已经跑通了多租户 + 数据隔离的模式
- Next.js + PostgreSQL (Supabase) 可以快速做出可演示的产品
- AI 辅助推荐已经可以在不增加人力的情况下做个性化服务

### 你在这个行业有真实积累

- 深度体验过 25 个本、35 场游戏，有真实的行业体感
- 写剧本杀内容，知道玩家的真实痛点是什么
- 懂技术落地，可以直接和开发者对接

## 三、我们解决什么问题

### 店家的痛苦

| 问题 | 现状 | 我们的方案 |
|------|------|-----------|
| 排期靠脑子 | 排一个本要翻半天表 | 日历拖拽排期，冲突自动检测 |
| 卡司被用废 | 连轴转没人管 | 卡司疲劳预警 + 工作量统计 |
| 道具管理混乱 | 丢了不知道谁用的 | 按排期记录领用、盘点、损耗 |
| 成本算不清 | 不知道一场本赚了多少 | 实时成本核算（工资+道具+房租分摊） |
| 客人靠微信拼本 | 效率低、体验差 | 客人自主查看排期、选角色、扣费报名 |

### 卡司/DM 的痛苦

| 问题 | 现状 | 我们的方案 |
|------|------|-----------|
| 没有晋升通道 | 干几年就转行 | DM 成长体系（学徒→金牌），等级对应收入 |
| 不知道排班 | 每天等通知 | 个人排班日历，随时查看 |
| 不知道收入 | 算不清自己拿了多少 | 薪资明细（基础+提成），每月可查 |
| 想休假不知道找谁 | 微信发消息不一定批 | 申请休假流程，透明审批 |
| 有本事不知道怎么变现 | 只能靠口碑 | 改本版税系统，作品上传→店家付费使用 |

### 发行商的痛苦

| 问题 | 现状 | 我们的方案 |
|------|------|-----------|
| 不知道本卖得怎么样 | 全靠店家反馈 | 实时销售数据，哪个城市/哪个店/哪个本 |
| 不知道玩家真实反馈 | 全靠豆瓣/小红书 | 评价数据从"文笔/流程/氛围"三维度拆解 |
| 不知道本被改成什么样 | 完全不知道 | 店家改本记录汇总，知道哪些本被改得最多 |
| 新本不知道推给谁 | 群发广告 | 精准推送给相关店家（根据历史偏好） |

## 四、核心壁垒

> 这个系统最难被复制的部分是什么？

### 第一层：DM 成长体系认证

行业第一套 DM 职业化标准。

```
学徒（跟训考证）→ 初级DM（能带本）→ 正式DM（优先排班）→ 高级DM（高提成）→ 金牌DM（可培训新人）
```

当 DM 习惯了这套成长体系，**他们就会留在这个平台**——就像程序员留在 GitHub 而不是自建 Git 服务器。

### 第二层：全产业链数据

同时拥有发行商数据（哪些本卖得好）、店家数据（哪个店用得最多）、卡司数据（哪个 DM 带得好）、玩家数据（哪个本受欢迎）——

**这个数据护城河一旦建立，没有人能绕开你。**

### 第三层：卡司改本生态

让有能力的卡司把自己的改本记录上传平台，店家付费使用。这不只是增收工具，这是**让整个行业的内容创作者都留在你这里**。

## 五、商业模式

### 第一阶段：SaaS 订阅（0 → 500 家店）

| 套餐 | 价格 | 功能 |
|------|------|------|
| 基础版 | 299/月 | 排期管理、卡司管理、房间管理、客人端报名 |
| 专业版 | 599/月 | 基础版 + 物资管理、卡司任务、DM 成长体系 |
| 旗舰版 | 999/月 | 专业版 + 成本核算、客诉管理、数据报表、优先客服 |

### 第二阶段：连锁店定制 + 发行商数据服务（500 → 2000 家店）

- 连锁品牌定制部署，按店数收费
- 发行商数据订阅（发行商付费看行业数据）

### 第三阶段：平台化（2000+ 家店）

- 打通美团/大众点评/猫眼（他们需要你的实时运营数据）
- 周边商品 + 伴手礼分销（从卡司原创周边切入）
- 晚宴旅游联动（"打完本去剧本背景地旅游"）

## 六、推广策略

### 第一步：单点验证（0 → 10 家店）

- 找 5-10 家认识的店，免费用三个月
- 目标：验证产品有没有人用、功能有没有解决痛点
- 同步做 DM 成长体系的内测

### 第二步：口碑扩散（10 → 100 家店）

- 做出 DM 认证体系，让卡司主动帮你推广
- 在剧本杀内容账号（抖音/B站）投放 demo 视频
- 找行业大V联合开发（你说想找剧本杀陪跑大V）

### 第三步：规模化（100 → 500 家店）

- 建立地推团队，扫当地剧本杀店
- 发行商联动（发行商推荐给他们的合作店家）
